# Markdown 书写规则

这套东华论文生成脚本建议遵守下面这套 `md` 书写规则。

## 1. 文档总体结构

- 第一条一级标题 `#` 默认作为论文总题目。
- 正文中的一级、二级、三级标题分别使用 `#`、`##`、`###`。
- `# 摘要` 或 `## 摘要` 会写入中文摘要区。
- `# ABSTRACT` 或 `## ABSTRACT` 会写入英文摘要区。
- `# 参考文献` 之后的内容会写入参考文献区。
- `# 致谢`、`# 附录`、`# 外文原文及译文` 会写入对应章节。

示例：

```md
# 可降解类玻璃高分子材料结构设计与性能研究

# 一、绪论

## （一）研究背景

正文内容……

# 参考文献

[1] ...
```

## 2. 正文引用

- 正文里的 `[1]`、`[1,4]`、`[1-3]` 会自动转成上标。
- 推荐直接写在正文句子里，不要手工再加上标符号。

示例：

```md
热固性材料难以回收[1]，因此动态共价网络受到关注[1,4]。
```

## 3. 表格写法

- 使用标准 Markdown 表格。
- 表格标题单独写在表格上方一行，格式为 `表题：标题内容`。
- 不要手工写 `表2-1` 这样的编号，脚本会按章节自动编号。
- 默认不自动生成 `续表`；长表是否跨页取决于 Word / WPS 的真实分页结果。

示例：

```md
表题：测试组分数据
| 组分 | Hf(kcal/mol) | Sf(kcal/mol) | Cp(kcal/mol) |
| --- | --- | --- | --- |
| A1 | 100 | 100 | 100 |
| A2 | 110 | 120 | 130 |
```

## 4. 图片写法

- 使用标准 Markdown 图片语法。
- 图片标题优先写在图片上一行，格式为 `图题：标题内容`。
- 如果没写 `图题：`，则会优先使用 `![这里的文字](图片路径)` 里的文字作为图题。
- 不要手工写 `图2-1`，脚本会按章节自动编号。
- 图片路径支持绝对路径，也支持相对当前 `md` 文件的相对路径。

示例 1：

```md
图题：气缸压力变化曲线
![](./figures/pressure_curve.png)
```

示例 2：

```md
![气缸压力变化曲线](./figures/pressure_curve.png)
```

## 5. 摘要与关键词

- `# 摘要` 或 `## 摘要` 进入中文摘要区。
- `关键词：` 这一行会被识别为中文关键词。
- `# Abstract` 或 `## Abstract` 进入英文摘要区。
- `KEY WORDS:` 这一行会被识别为英文关键词。
- 关键词标签可以加粗，例如 `**关键词**：` 和 `**KEY WORDS**:`。

示例：

```md
# 摘要

这里是中文摘要。

关键词：类玻璃高分子，PDK，PCL

# Abstract

This is the English abstract.

KEY WORDS: vitrimer, PDK, PCL
```

## 6. 当前建议

- 文件可以使用 UTF-8 或 UTF-8 with BOM 编码。
- 图、表、正文之间最好各空一行，便于解析稳定。
- 表格单元格内容尽量保持单行短文本。
- 普通正文中的反引号和加粗标记会自动去掉，例如 `` `PDMS` `` 会写成 `PDMS`。
- 如果后续学校要求必须精确写出 `续表`，这一步需要放到 Word / WPS 完成真实分页之后再处理。
- 目录仍建议在 Word / WPS 中手动更新。
