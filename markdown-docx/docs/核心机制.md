# 核心机制

本项目用于把 Markdown 论文稿写入东华大学本科毕业论文 Word 母版，输出符合模板结构的 `.docx` 文件。

## 工作流程

1. 读取 `outputs/dhu_undergrad_clean_template.docx` 作为 Word 母版。
2. 解析 Markdown 中的题目、摘要、关键词、正文标题、段落、图、表、参考文献等内容。
3. 根据 `dhu_template_schema.json` 中的锚点定位封面、摘要、目录、正文、参考文献、致谢、附录等区域。
4. 清空母版中的占位内容，保留原模板的页面结构、分节、页眉页脚、页边距和基础样式。
5. 将解析后的论文内容写入对应区域。
6. 保存为新的 Word 文档。

## 模板直填机制

项目不会从零创建一个 Word 模板，而是复用已有干净母版。这样可以最大程度保留学校模板中的：

- 封面结构
- 学术诚信声明和版权授权页
- 页眉页脚
- 页面边距
- 分节结构
- 页码设置
- 标题、正文、图题、表题、参考文献等段落格式

脚本只负责替换内容和补充正文结构。

## Markdown 解析机制

Markdown 会被解析为内部内容块，包括：

- `heading`：正文标题
- `paragraph`：正文段落
- `table`：Markdown 表格
- `image`：Markdown 图片

特殊章节会写入模板中的固定区域：

- `# 摘要` 或 `## 摘要`：中文摘要
- `# ABSTRACT` 或 `## ABSTRACT`：英文摘要
- `# 参考文献`：参考文献
- `# 致谢`：致谢
- `# 附录`：附录
- `# 外文原文及译文`：外文原文及译文

第一条非特殊一级标题默认作为论文总题目。

## 编号机制

正文标题由脚本重新编号：

- 一级标题生成 `1  标题`
- 二级标题生成 `1.1 标题`
- 三级标题生成 `1.1.1 标题`

图表编号按当前章编号生成：

- 表格生成 `表1-1 表题`
- 图片生成 `图1-1 图题`

用户在 Markdown 中不需要手动写图表编号。

## 样式机制

写入 Word 时，脚本会复用模板中已有段落作为格式原型，并结合 `legacy_format_rules.py` 设置：

- 标题字体、字号、加粗、居中
- 正文字体、字号、首行缩进、行距
- 图题和表题格式
- 表格单元格格式
- 参考文献悬挂缩进
- 中英文关键词格式

## 输出效果

生成文档包含以下主要区域：

- 封面
- 学术诚信声明
- 版权使用授权书
- 中文摘要和关键词
- 英文摘要和关键词
- 目录页
- 正文章节
- 图和表
- 参考文献
- 致谢、附录、外文原文及译文

目录域需要在 Word 或 WPS 中打开后手动更新。
